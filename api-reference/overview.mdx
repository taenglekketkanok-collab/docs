The Google REST API gives you programmatic access to all platform capabilities — search, account management, and webhook configuration. All requests go over HTTPS, accept and return JSON, and authenticate with Bearer tokens.

## Base URL

```
https://api.google.com/v1
```

All endpoint paths in this reference are relative to this base URL.

## Request format

All API requests must include:

| Header | Value |
|---|---|
| `Authorization` | `Bearer YOUR_API_KEY` |
| `Content-Type` | `application/json` (for POST/PATCH requests) |

Example:

```http
POST /v1/accounts HTTP/1.1
Host: api.google.com
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json

{ "name": "Acme Corp", "email": "admin@acme.com" }
```

## Response format

All responses return JSON. Successful responses have HTTP `2xx` status codes. Errors follow a standard structure:

```json
{
  "error": {
    "code": "not_found",
    "message": "Resource was not found.",
    "status": 404,
    "request_id": "req_01jkl012"
  }
}
```

## Versioning

The current API version is **v1**. Backward-incompatible changes will be released as a new major version (e.g., `/v2`). The previous version remains available for at least 12 months after a new major version is released.
