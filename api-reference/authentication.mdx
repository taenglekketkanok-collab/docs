Every request to the Google API must include a valid API key in the `Authorization` header. The key is passed as a Bearer token and must carry the correct scope for the operation you are performing.

## Authorization header

```http
Authorization: Bearer YOUR_API_KEY
```

Replace `YOUR_API_KEY` with the key you generated in **Settings → API Keys**.

## Scope requirements

Each endpoint requires a specific scope. Your API key must have been created with the relevant scope:

| Endpoint | Required scope |
|---|---|
| `GET /search` | `search:read` |
| `GET /accounts` | `accounts:read` |
| `POST /accounts` | `accounts:write` |
| `GET /accounts/{id}` | `accounts:read` |
| `PATCH /accounts/{id}` | `accounts:write` |
| `GET /webhooks` | `webhooks:manage` |
| `POST /webhooks` | `webhooks:manage` |
| `DELETE /webhooks/{id}` | `webhooks:manage` |

## Authentication errors

| HTTP Status | Error Code | Description |
|---|---|---|
| `401` | `unauthorized` | No API key provided, or the key is invalid |
| `401` | `key_expired` | The API key has expired |
| `403` | `insufficient_scope` | The key does not have the required scope |

Example `401` response:

```json
{
  "error": {
    "code": "unauthorized",
    "message": "Invalid API key. Verify your key in Settings → API Keys.",
    "status": 401,
    "request_id": "req_01jkl012"
  }
}
```
